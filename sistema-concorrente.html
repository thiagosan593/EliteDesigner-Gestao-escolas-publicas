Análise de Sistema Concorrente:

Plataforma de ensino Ânima Educação



01 - Visibilidade do status do sistema:

Se o usuário entrar de um determinado serviço por engano, ele saberá onde está, podendo voltar a etapa anterior e fará com que ele se sinta menos frutado caso tenha que voltar o fluxo desde o início, pois a facilidade de voltar etapas anteriores fará que o usuário não perca tempo e nem energia, além de aumentar a eficiência do sistema resultando em uma experiência satisfatória.

02- Correspondência entre o sistema e o mundo real.

Os símbolos escolhidos são símbolos comuns no dia a dia, o que não fará o usuário pensar para descobrir o que é aquela tarefa. Quando o usuário passa o cursor sobre cada ícone, ele apresenta o nome do ícone, melhorando mais ainda a sua usabilidade. Tendo em vista que, cada ícone é implementado com uma gama de conteúdo em seu interior.

03- Consistência e padrões

O calendário de aula geral e o calendário de aula individual mantêm a consistência e o mesmo padrão de uma página para a outra. As cores também seguem um padrão, como a fonte de letras, botões e novas abas, o sistema em sua totalidade tem uma apresentação bem homogênea.

04- Restrições

O sistema não permite o envio de uma mensagem sem assunto. O mesmo também é restritivo quanto ao envio de arquivos, o usuário não poderá efetuar essa ação sem anexar o documento no formato exigido no rodapé, e também não poderá enviar quando se encerra o prazo de entrega, o sistema ele não permite.

05- Feedback

Após o envio de uma tarefa no sistema, o usuário recebe o feedback sobre a mesma como, data e hora de envio, bem como o status de avaliação sobre a tarefa. E aviso sobre aula online, entregando uma aba para que o aluno entre na sala virtual. As avaliações online também dá feedback, quando realizadas se o aluno tentar fazer novamente o sistema não permite avisando-o com uma que tal atividade já foi realizada.

Percepção geral:

Com base em análises, o sistema é forte em usabilidade e facilidade de uso, sendo muito intuitivo, pois não somente a visibilidade do status do sistema permitindo a fácil navegação, bem como a correspondência entre o sistema e o mundo real é bem implementada,a padronização geral, das cores, ícones, fontes também contribuem para uma ótima experiência, os mecanismos de feedback fornecem informações úteis, contribuem para uma experiência positiva do usuário e o ajudam a usar o sistema de forma fácil.
